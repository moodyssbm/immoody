Step 0: Overview

This guide will help you to set up a DOS-like pause command for macOS (OS X if you're on an older version) and Linux. Keep in mind that anything that looks like this is code. I will specify whether the code being displayed is for the terminal or for a file.

Requirements:

  • A computer running macOS/OS X (You can follow this guide for Linux. Just use apt instead of brew.)
  • Homebrew for macOS. Install guide.
  • Node - Install this after Homebrew.
    • brew install node

Step 1: Folder Setup

You can really put this anywhere. We're just going to be referencing this folder later, so remember where it is! Once you have decided where to put your folder, navigate to it using a terminal cd path/to/your/folder and initialize node npm init

Node will spit a lot of things out to the console. Just keept hitting Enter/Return until you're back to the main shell.

Install the "keypress" package using Node Package Manager npm i keypress

While you have your terminal open in the root of your new folder, initialize the main JavaScript file you'll be editing. I call mine pause touch pause && chmod +x pause.js

I'll get to why I didn't add a .js extension in the next step.

Step 2: The Code

Go ahead and copy + paste the following code into your pause file you created at the end of step 1.

  • #!/usr/bin/env node
  • let keypress = require('keypress');
  • keypress(process.stdin);
  • console.log('Press any key to continue . . .');
  • process.stdin.on('keypress',
    • function(ch,key)
    • {
      • process.exit();
    • }
  • );
  • process.stdin.setRawMode(true);

So, we named the file pause instead of pause.js because we don't want to call system("pause.js"); in our C code. Dropping the extension just allows us to call pause. Now, this is possible because we set our environment on the first line to execute this file with node.

Step 3: Test the Code

If you've followed the guide to a T up to this point, you should be able to execute your program now! Try it in your terminal with ./pause

If it's running, then great! You should be able to press any key while in the terminal to end the node execution. If you're getting an error, I'd recommend going through the guide again to see if you missed any steps.

Step 4: Making it a Global Executable

Just to preface this step, if you've already set up a local ~/bin folder for your User account in MacOS, just move your pause executable there and install the keypress package again to your ~/bin folder. Otherwise, continue reading.

Since you should already be in your terminal, navigate to your home folder and make your bin folder with
cd ~ && mkdir bin. Next, go ahead and copy the node_modules folder and pause file from the last directory to ~/bin. Now you'll want to navigate back to your home folder with cd ~.

Go ahead and open your .bash_profile file in your home directory with nano ./.bash_profile. Nano--a terminal-based text editor--will open with your .bash_profile file. All you have to do here is add this line to the file to specify your User executable path. PATH=$PATH:$HOME/bin This will make any executable file in your ~/bin folder accessible from anywhere.

Step 5: Final Steps and Testing

Go ahead and grab my pause-test Python executable from here. Once it finishes downloading, decompress the .zip file by double clicking on it. Now you should have a folder called "pause-test". Go into the folder and double click on the pause-test executable. A terminal window should appear, and it should be waiting for you to press any key.
